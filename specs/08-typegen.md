# Typegen

> Generate TypeScript types from live MCP tool schemas so `call()` gets typed arguments and return values.

## Motivation

The JS API (`07-js-api.md`) gives you `call(name: string, args?: Record<string, unknown>)` — fully untyped. If you call `filesystem/read_file`, you get no autocomplete on `args` and no type on the result. You're on your own.

MCP tools already declare their shapes via `inputSchema` (required) and `outputSchema` (optional) as JSON Schema objects. Typegen introspects the running daemon, converts those schemas to TypeScript types, and produces a type map that makes `call()` fully typed — autocomplete on tool names, typed arguments, typed results.

This is the same model as GraphQL codegen or Prisma: you run a command when your schema changes, and your editor lights up with types. No runtime cost — just a `.d.ts` file.

## CLI Command

```
mcpd typegen [--output <path>] [--config <path>]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--output`, `-o` | `mcpd.d.ts` | Output file path |
| `--config`, `-c` | (default resolution) | Path to `mcpd.config.json` |

The command:

1. Ensures the daemon is running (auto-starts if needed, same as all CLI commands)
2. Calls `tools/list` to fetch all tools from all servers
3. Converts each tool's `inputSchema` and `outputSchema` to TypeScript types
4. Writes a `.d.ts` file with the tool map and a typed `createClient` overload

```bash
# Generate types
mcpd typegen

# Custom output path
mcpd typegen -o src/types/mcpd.d.ts
```

## Generated Output

For a daemon with two servers (`filesystem` with `read_file`/`write_file`, `github` with `create_issue`):

```ts
// mcpd.d.ts
// Auto-generated by `mcpd typegen` — do not edit
// Generated at: 2026-02-21T12:00:00.000Z

import type { McpdClient, CreateClientOptions } from 'mcpd';

export interface McpdToolMap {
  'filesystem/read_file': {
    input: {
      path: string;
      encoding?: string;
    };
    output: unknown;
  };
  'filesystem/write_file': {
    input: {
      path: string;
      content: string;
    };
    output: unknown;
  };
  'github/create_issue': {
    input: {
      repo: string;
      title: string;
      body?: string;
      labels?: string[];
    };
    output: {
      number: number;
      url: string;
    };
  };
}

export type McpdToolName = keyof McpdToolMap;

/**
 * Creates a typed mcpd client. Tool names, arguments, and return types
 * are all inferred from the generated tool map.
 */
export declare function createTypedClient(
  options?: CreateClientOptions
): Promise<TypedMcpdClient>;

export interface TypedMcpdClient extends McpdClient {
  call<K extends McpdToolName>(
    name: K,
    args: McpdToolMap[K]['input'],
    options?: import('mcpd').CallOptions
  ): Promise<McpdToolMap[K]['output']>;

  // Fallback for tools not in the map (added after typegen, etc.)
  call(
    name: string,
    args?: Record<string, unknown>,
    options?: import('mcpd').CallOptions
  ): Promise<import('mcpd').CallResult>;
}
```

### Design Decisions

**`output` defaults to `unknown`.** Most MCP tools don't declare `outputSchema` — they return unstructured `content` arrays. When `outputSchema` is absent, the output type is `unknown`, which is honest. Users can narrow it themselves or wait for servers to adopt structured output.

**`createTypedClient` is a thin wrapper.** It calls the real `createClient()` and returns it cast to `TypedMcpdClient`. No runtime overhead — the types are erased at compile time. The generated file provides the wrapper so users get types with a single import.

**Overloaded `call()`.** The typed overload handles known tools; the fallback accepts any `string` for tools added after typegen ran. This means stale types degrade gracefully — you lose autocomplete but nothing breaks.

## JSON Schema to TypeScript Conversion

Use `json-schema-to-typescript` (the standard library for this, 15M+ weekly downloads). It handles:

- `type: "string" | "number" | "integer" | "boolean" | "null"` → TS primitives
- `type: "object"` with `properties` / `required` → interfaces with optional fields
- `type: "array"` with `items` → `T[]`
- `enum` → union literals
- `oneOf` / `anyOf` → union types
- `allOf` → intersection types
- `$ref` → resolved inline (MCP tool schemas rarely use `$ref`)
- Nested objects → nested interfaces

### Edge Cases

| JSON Schema | TypeScript | Notes |
|---|---|---|
| No `outputSchema` | `unknown` | Most tools today |
| `properties: {}` or missing `properties` | `Record<string, unknown>` | Accept anything |
| `additionalProperties: true` | `{ [k: string]: unknown; ...named }` | Indexed signature + named props |
| Unsupported keywords | Ignored, falls back to `unknown` | Graceful degradation |
| Empty `inputSchema` (no `properties`, no `required`) | `Record<string, never>` | Tool takes no arguments |

### Naming

Types are generated inline in the tool map — no separate named interfaces. This avoids name collisions (two servers could have tools with the same name) and keeps the output flat. The tool map key (`server/tool`) is the unique identifier.

## Typed Client Integration

The generated types plug into the existing `McpdClient` from `07-js-api.md`. Two approaches, both supported:

### Approach 1: Use `createTypedClient` (simple)

```ts
import { createTypedClient } from './mcpd.js';

const mcpd = await createTypedClient();

// Fully typed — autocomplete on tool name, args, and result
const result = await mcpd.call('filesystem/read_file', { path: '/etc/hostname' });
```

### Approach 2: Cast the existing client (manual)

```ts
import { createClient } from 'mcpd';
import type { TypedMcpdClient } from './mcpd.js';

const mcpd = await createClient() as TypedMcpdClient;
const result = await mcpd.call('filesystem/read_file', { path: '/etc/hostname' });
```

Both are equivalent at runtime. Approach 1 is syntactically cleaner.

## Implementation

### Dependencies

Add `json-schema-to-typescript` as a regular dependency (it's needed at CLI runtime for the `typegen` command, not just at build time).

### File Structure

```
src/
```

### `src/codegen/typegen.ts`

```ts
import { compile } from 'json-schema-to-typescript';

type ToolEntry = {
  server: string;
  tool: {
    name: string;
    inputSchema: Record<string, unknown>;
    outputSchema?: Record<string, unknown>;
  };
};

type TypegenOptions = {
  /** Timestamp for the generated header. */
  generatedAt?: string;
};

export async function generateTypes(
  tools: ToolEntry[],
  options?: TypegenOptions
): Promise<string> {
  const toolTypes: string[] = [];

  for (const { server, tool } of tools) {
    const qualifiedName = `${server}/${tool.name}`;
    const inputType = await schemaToType(tool.inputSchema, `${qualifiedName}_input`);
    const outputType = tool.outputSchema
      ? await schemaToType(tool.outputSchema, `${qualifiedName}_output`)
      : 'unknown';

    toolTypes.push(`  '${qualifiedName}': {\n    input: ${inputType};\n    output: ${outputType};\n  };`);
  }

  const timestamp = options?.generatedAt ?? new Date().toISOString();

  return `// Auto-generated by \`mcpd typegen\` — do not edit
// Generated at: ${timestamp}

import type { McpdClient, CreateClientOptions, CallOptions, CallResult } from 'mcpd';

export interface McpdToolMap {
${toolTypes.join('\n')}
}

export type McpdToolName = keyof McpdToolMap;

export declare function createTypedClient(
  options?: CreateClientOptions
): Promise<TypedMcpdClient>;

export interface TypedMcpdClient extends McpdClient {
  call<K extends McpdToolName>(
    name: K,
    args: McpdToolMap[K]['input'],
    options?: CallOptions
  ): Promise<McpdToolMap[K]['output']>;

  call(
    name: string,
    args?: Record<string, unknown>,
    options?: CallOptions
  ): Promise<CallResult>;
}
`;
}

async function schemaToType(
  schema: Record<string, unknown>,
  name: string
): Promise<string> {
  // Use json-schema-to-typescript to convert, then extract the type body
  // (strip the generated interface wrapper, we inline it)
  const compiled = await compile(schema, name, {
    bannerComment: '',
    additionalProperties: false,
    unknownAny: true,
  });

  // Extract just the type body from the generated interface
  return extractTypeBody(compiled);
}

function extractTypeBody(compiled: string): string {
  // json-schema-to-typescript produces:  export interface Foo { ... }
  // We extract the { ... } part for inlining into the tool map
  const match = compiled.match(/\{[\s\S]*\}/);
  if (!match) return 'Record<string, unknown>';
  return match[0];
}
```

### `src/cli/commands/typegen.ts`

```ts
import { sendRequest, ensureDaemon } from '../client.js';
import { generateTypes } from '../../codegen/typegen.js';
import fs from 'node:fs';
import path from 'node:path';

export async function typegenCommand(options: { output: string; config?: string }) {
  await ensureDaemon(options.config);

  const tools = await sendRequest('tools/list');

  const content = await generateTypes(tools as ToolEntry[]);

  const outputPath = path.resolve(options.output);
  fs.writeFileSync(outputPath, content, 'utf-8');

  console.log(`Generated ${(tools as unknown[]).length} tool types → ${outputPath}`);
}
```

### `createTypedClient` Runtime

The generated `.d.ts` declares `createTypedClient` but doesn't implement it — it's a type-level wrapper. The actual runtime implementation lives in the `mcpd` package:

```ts
// src/client/index.ts — add to existing client module
export async function createTypedClient(options?: CreateClientOptions): Promise<McpdClient> {
  // Same as createClient — the types are purely compile-time
  return createClient(options);
}
```

The function signature in the generated `.d.ts` overrides the return type to `TypedMcpdClient`, giving typed `call()`.

## Testing

### Unit Tests (`src/codegen/typegen.test.ts`)

```ts
describe('generateTypes', () => {
  it('generates types for tools with inputSchema', async () => {
    const tools = [
      {
        server: 'fs',
        tool: {
          name: 'read',
          inputSchema: {
            type: 'object',
            properties: { path: { type: 'string' } },
            required: ['path'],
          },
        },
      },
    ];
    const output = await generateTypes(tools);
    expect(output).toContain("'fs/read'");
    expect(output).toContain('path: string');
    expect(output).toContain('output: unknown');
  });

  it('generates typed output when outputSchema present', async () => {
    const tools = [
      {
        server: 'gh',
        tool: {
          name: 'create_issue',
          inputSchema: {
            type: 'object',
            properties: { title: { type: 'string' } },
            required: ['title'],
          },
          outputSchema: {
            type: 'object',
            properties: { number: { type: 'integer' }, url: { type: 'string' } },
            required: ['number', 'url'],
          },
        },
      },
    ];
    const output = await generateTypes(tools);
    expect(output).toContain('number: number');
    expect(output).toContain('url: string');
  });

  it('handles tools with no properties', async () => {
    const tools = [
      {
        server: 'srv',
        tool: {
          name: 'ping',
          inputSchema: { type: 'object' },
        },
      },
    ];
    const output = await generateTypes(tools);
    expect(output).toContain("'srv/ping'");
  });

  it('handles empty tool list', async () => {
    const output = await generateTypes([]);
    expect(output).toContain('interface McpdToolMap');
  });

  it('escapes special characters in tool names', async () => {
    const tools = [
      {
        server: 'my-server',
        tool: {
          name: "tool's",
          inputSchema: { type: 'object' },
        },
      },
    ];
    const output = await generateTypes(tools);
    expect(output).toContain("my-server/tool\\'s");
  });
});
```
